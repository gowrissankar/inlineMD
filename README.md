# inlineMD

inlineMD is a Visual Studio Code extension that implements an interactive editing interface directly within the Markdown preview surface. By double-clicking any rendered block element, users can view and edit the corresponding raw Markdown source code inside a VS Code-native inline peek widget.

## Architecture and Technical Decisions

The extension is designed around VS Code's Custom Editor API to provide a unified editing experience, bypassing the friction of side-by-side split panels.

```
┌────────────────────────────────────────────────────────┐
│                   Extension Host (Node.js)             │
│                                                        │
│   ┌──────────────┐      ┌─────────────────────────┐    │
│   │  TextDocument│◄────►│  Custom Editor Provider │    │
│   │  (in-memory) │      │  - Queries user fonts   │    │
│   └──────────────┘      │  - WorkspaceEdit API    │    │
│                         └──────────┬──────────────┘    │
└────────────────────────────────────┼───────────────────┘
                                     │ postMessage (handshake)
┌────────────────────────────────────▼───────────────────┐
│                   Webview (Browser Context)            │
│                                                        │
│   ┌──────────────────────────────────────────────┐     │
│   │  markdown-it + highlight.js (Extension Side) │     │
│   │  - Line attribute injection ([data-line])    │     │
│   └──────────────────────────────────────────────┘     │
│   ┌──────────────────────────────────────────────┐     │
│   │  Interactive View Layer                      │     │
│   │  - Node resolution on double-click           │     │
│   │  - Inline Peek Widget (<textarea>)           │     │
│   └──────────────────────────────────────────────┘     │
└────────────────────────────────────────────────────────┘
```

### Webview Sandboxing and Security Constraints
Standard VS Code extensions run inside a sandboxed Extension Host process, separate from the main VS Code workbench thread. The webview itself is isolated in an iframe with no access to Node.js or the local file system.

To edit files securely, the webview captures edit events and routes them via JSON messages (`postMessage`) to the Custom Editor Provider running on the Extension Host. The Host then applies changes using VS Code's formal API layer.

### Abstract Syntax Tree (AST) Line Mapping
Because the Markdown parser (`markdown-it`) processes source code into an AST, we can inspect block-level tokens before they compile to HTML.

We injected a custom plugin rule into the parsing pipeline that reads the token's source map boundaries (`token.map`) and stamps them onto the emitted HTML tags as dataset properties:
```html
<p data-line-start="12" data-line-end="14">Rendered paragraph content...</p>
```
On a double-click event, the webview script traverses the DOM tree upwards, resolves the targeted line range, and requests those specific lines from the document buffer.

### Workspace Configuration Synchronization
To achieve visual consistency with the native editor, the extension dynamically queries the active VS Code workspace configurations for the following user settings:
- `markdown.preview.fontFamily`
- `markdown.preview.fontSize`
- `markdown.preview.lineHeight`

These values are injected into the HTML template header as root-level CSS properties. Combined with the default styles extracted from the VS Code core repository, this renders the view identical to the built-in preview.

### Cross-Site Scripting (XSS) Mitigation
To protect users from malicious Markdown documents containing embedded scripts, the extension implements a strict Content Security Policy (CSP) combined with single-use cryptographic nonces. Only scripts and style blocks generated by the extension carrying the correct nonce are permitted to execute. Raw HTML rendering is disabled in the markdown-it configuration.

## Key Features

- **Unified Surface:** Editing and previewing are combined on a single canvas, eliminating split-tab layout management.
- **Native Widget Styling:** The inline peek editor replicates the color schema, borders, and margins of VS Code's built-in reference peek views.
- **Integrated Undo Stack:** Edits are committed via `vscode.WorkspaceEdit`, automatically integrating transactions into VS Code's native undo/redo history.
- **Syntax Highlighting:** Embedded code blocks are parsed using `highlight.js` matching VS Code's default theme selectors.

## Limitations and Future Scope

- **Block-level Granularity:** Source mappings are resolved at the block level (headings, paragraphs, lists, table elements). Inline elements (bold, italicized runs) map back to their containing block.
- **Document Navigation:** Future updates will resolve webview navigation for local Markdown file links, routing document lookup directly through the custom provider.
- **Extended Render Engines:** Integration of plugins for Mermaid diagram formatting (`markdown-it-mermaid`) and mathematical notation (`markdown-it-katex`).
